Museum Exhibition Classification

YOLO Object Detection for Museum Exhibition Classification

Introduction

In this project, we focus on building an object detection pipeline for classifying museum exhibits using the YOLO (You Only Look Once) model. This model allows real-time object detection and classification, which can be highly effective in environments like museums where multiple objects or artworks need to be quickly identified and categorized.

The implementation of this project involves building and running models using TensorFlow Lite, processing camera input, handling raw data, and optimizing performance to ensure accurate results. Let’s dive into the key aspects and challenges of the project.

Project Overview

Motivation

Museums contain various exhibits that require careful classification and tagging. Manually cataloging every piece of art or exhibit can be both time-consuming and error-prone. Using computer vision techniques, such as object detection, helps in automating this process, making it both quicker and more accurate.

The main goal of this project is to use the YOLO model to detect objects in real-time through a camera, classify them, and extract meaningful metadata, allowing for efficient and automated tagging of exhibits.

Project Structure

Core Components

  1. YOLO Model Loading:
    The YOLO model is loaded into the app using TensorFlow Lite. It’s a highly efficient object detection model that can work in real-time, making it suitable for mobile applications.

  2. Camera Input:
    The app uses the device camera to capture images in real-time. The CameraViewModel handles camera permission checks, video sessions, and captures, ensuring that the data is fed properly to the object detection pipeline.

  3. Data Preprocessing:
    Before feeding the image data to the YOLO model, we resize and normalize the image. This ensures that the model receives data in a consistent format and improves the accuracy of object detection.

  4. Object Detection and Classification:
    Once the preprocessed image data is passed to the YOLO model, it outputs an array of bounding boxes and class probabilities. These values are parsed, and the objects are classified into one of the predefined categories (e.g., "obj1", "obj2", …, "obj7").

  5. Anti-Reflection Model:
    To improve detection accuracy, a separate anti-reflection model processes images before they are fed into the YOLO model. This helps mitigate issues caused by glass reflections or glare, which are common in museum environments.

  6. Debugging Output:
    To analyze and optimize the model’s performance, we print and visualize bounding boxes, confidence scores, and their respective classifications. Raw data is printed for inspection, making it easier to diagnose and solve issues with the model’s output.

Key Challenges

  1. Bounding Box Parsing:
    The YOLO model outputs bounding boxes with associated probabilities for various classes. A critical part of the project was iterating through these bounding boxes, identifying the class with the highest probability, and correctly mapping it to the corresponding label.

  2. Data Handling and Debugging:
    Understanding the raw output of the YOLO model in terms of Float32 values was crucial for debugging. Extracting the correct data and handling large arrays of floats required careful inspection and verification.

  3. Reflection Mitigation:
    In museums, many exhibits are behind glass, which can cause reflections and interfere with the model’s accuracy. A separate anti-reflection model was introduced to enhance the detection performance.

  4. Performance Optimization:
    Ensuring real-time object detection is no small feat. To achieve high FPS (frames per second) and accurate detection, preprocessing was optimized, and model inference was streamlined to reduce any potential lags.

Lessons Learned

  1. Proper Data Handling: Correctly parsing and inspecting raw data is key in ensuring the accuracy of the model output.
  2. Performance Matters: With real-time object detection, performance optimizations in both preprocessing and inference are critical.
  3. Reflection Handling: Addressing environmental issues like reflections can greatly improve model accuracy, especially in unique settings like museums.

Conclusion

This project is an exciting step toward automating the classification of museum exhibits. By leveraging the YOLO model’s real-time detection capabilities and addressing environmental factors like reflections, we aim to make cataloging exhibits a seamless and efficient process.

Stay tuned for more updates as we continue to improve and fine-tune the model for better performance and accuracy!

Future Work

  1. Model Fine-Tuning: Experimenting with different versions of the YOLO model for better accuracy and performance.
  2. Better Reflection Handling: Developing more advanced anti-reflection techniques to further reduce environmental interference.
  3. Multi-Object Detection: Optimizing for detecting and classifying multiple objects within the same frame.
